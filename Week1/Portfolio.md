# COIT12206 – TCP/IP Protocols
## Markdown Guide for GitHub Portfolios

This document demonstrates common Markdown formatting used on GitHub.

---

# 1. Headings

Use `#` symbols to create headings.

# Main Title
## Section Title
### Subsection Title
#### Smaller Heading

---

# 2. Bullet Lists

Use `-` or `*`.

- Create GNS3 project
- Connect PC1 and PC2
- Configure IP addresses
- Test connectivity using ping

---

# 3. Numbered Lists

Use numbers followed by a period.

1. Open GNS3
2. Add two VPCS devices
3. Connect them using Ethernet
4. Configure IP addresses

---

# 4. Bold and Italics

Bold text:

**Important networking command**

Italic text:

*This is a note for students*

---

# 5. Inline Code

Use single backticks for commands inside text.

Example:

Use the `ping` command to test connectivity.

---

# 6. Code Blocks (Important for Networking)

Use triple backticks for command blocks.

```bash
ip address show
ping 10.10.1.2
```

```
ip address show
ping 10.10.1.2
```

```
backtick test
```
